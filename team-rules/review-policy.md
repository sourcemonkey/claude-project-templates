# レビュー方針

## レビュー観点

1. **動くか**: テストが通るか、手動でも動くか。
2. **仕様通りか**: docs/ の仕様と一致しているか。
3. **読めるか**: 命名・構造が他のメンバーに伝わるか。
4. **壊さないか**: 既存機能への影響、N+1、トランザクション境界。
5. **テストが妥当か**: ハッピーパスだけでなく異常系もカバーしているか。

## Claude Code の自己レビュー

PR を出す前（あるいは「実装完了」と報告する前）に、Claude Code 自身が以下を確認する。

- [ ] 仕様（docs/）と差分の対応が取れている
- [ ] `bin/rails test` が green
- [ ] カバレッジが 80% 以上（`coverage/index.html` で確認）
- [ ] `bin/rubocop` で違反がない
- [ ] `bin/brakeman --no-pager` で High 警告がない
- [ ] 新規追加した routes が `bin/rails routes` で確認できる
- [ ] seeds 投入後に主要画面が 200 を返す
- [ ] README に追記が必要な手順があれば追記済み

これらが満たせない場合、完了報告せずに原因と対処案を提示する。
